# Schema Spine Index Fields Reference

This document explains the structure and purpose of each top-level field in `schema-spine-index.json`. The index serves as the queryable map for all contracts, invariants, metrics, and dependencies across the VM-constellation.

## Top-Level Fields

| Field | Type | Description | Usage |
|-------|------|-------------|-------|
| `indexVersion` | string | Semantic version of the index structure itself. | Allows tools to detect breaking changes in index format. |
| `generatedAt` | string | ISO 8601 UTC timestamp of generation. | Used by CI to verify freshness and trigger regeneration if stale. |
| `schemas` | object | Map of schema `$id` → metadata and field definitions. | Query to find required fields, ranges, and consumers of a schema. |
| `invariants` | object | Map of invariant name → definition and usage stats. | Query to resolve invariant ranges and where they are applied. |
| `metrics` | object | Map of metric name → definition and telemetry linkage. | Query to find telemetry envelopes and expected metric ranges. |
| `contracts` | object | Map of contract type → schema ref and requirements. | Query to validate generated contract cards and required fields. |
| `registries` | object | Map of registry type → ID pattern and maintenance info. | Query to validate registry entries and identify responsible repos. |
| `consumers` | object | Map of consumer repo → dependencies and outputs. | Query to find all repos affected by a schema change. |

## Nested Structures

### `schemas.<schemaId>`
- **`title`**: Human-readable name of the schema.
- **`version`**: Semantic version (e.g., `1.0.0`).
- **`tier`**: Default access tier (`public`, `vault`, `lab`).
- **`fields`**: Map of field name → type, range, and requirement status.
- **`consumedBy`**: List of repo names that import this schema.

### `invariants.<name>`
- **`definedIn`**: `$id` of the schema defining this invariant.
- **`type`**: Data type (e.g., `number`).
- **`range`**: Valid `[min, max]` array.
- **`semantics`**: Short description of what the invariant measures.
- **`usedInContracts`**: List of contract types using this invariant.
- **`usedInRegistries`**: List of registry types tracking this invariant.

### `consumers.<repoName>`
- **`type`**: Role of the repo (e.g., `vault-repo`, `lab-repo`).
- **`tier`**: Access tier of the repo.
- **`usesSchemas`**: Array of schema `$id`s imported.
- **`usesInvariants`**: Array of invariant names referenced.
- **`producesRegistries`**: Array of registry types maintained by this repo.

## Validation and Maintenance
- The index is generated by `hpc-generate-spine-index.py`.
- CI enforces that all `consumedBy` references correspond to actual repositories with `.horrorplace-contracts.json` declarations.
- Drift is detected when a consumer repo pins a schema version that no longer matches the `schemas.<id>.version` field.
